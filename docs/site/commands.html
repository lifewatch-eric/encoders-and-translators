01 — Prerequisites

Setup

The only required dependency is lxml. Python 3.9 or later is required. Java is optional (only needed for Saxon).

Install lxml

bash
$ pip install lxml

Clone or unzip the repository

bash — clone from GitHub
$ git clone https://github.com/your-org/lifewatch-xslt-transformations.git
$ cd lifewatch-xslt-transformations
bash — navigate to the transformation folder
$ cd transformations/EML211_to_EML220

# You should now be here:
# lifewatch-xslt-transformations/transformations/EML211_to_EML220/
All commands below assume you are inside transformations/EML211_to_EML220/ unless stated otherwise.

step 2

step 3

step 4

step 5

04 — Auto runner

run.py — Transform & Test in One Command

This is the main entry point. It scans for new input files, runs the XSLT, runs the test suite, and shows you exactly where to find the output.

Standard run — transform all new inputs

bash
$ python3 run.py

# What it does:
#  1. Scans examples/input/ for every .xml file
#  2. Skips files that already have a matching output
#  3. Transforms only NEW files → writes to examples/output/
#  4. Runs the full test suite on each new file
#  5. Prints the output path with a NEW marker

Force re-transform (overwrite existing outputs)

bash
$ python3 run.py --force

# Re-runs the transformation even when outputs already exist.
# Use this after editing the stylesheet (xslt/main.xsl).

Transform a single specific file

bash
$ python3 run.py --input examples/input/your-dataset-eml211.xml

# Works with any path, not just files inside examples/input/:
$ python3 run.py --input /Users/houdabensalah/Downloads/my-data.xml

Transform and save to a specific output path

bash
$ python3 run.py \
    --input  examples/input/your-dataset-eml211.xml \
    --output /Users/houdabensalah/Desktop/result-eml220.xml

Override the packageId in the output

bash
$ python3 run.py --package-id my.new.package.id.2026

Open the output folder automatically (macOS / Linux)

bash
$ python3 run.py --open-folder

# Opens examples/output/ in Finder (macOS) or Files (Linux) after the run.

Transform only — skip the test suite

bash
$ python3 run.py --no-tests

# Faster. Use when you only need the output file, not the validation.

All run.py flags at a glance

FlagShortDescriptionDefault
--input FILE-iTransform a single file instead of scanning input/scan all in examples/input/
--output FILE—Save result to a specific pathexamples/output/ (auto-named)
--force-fRe-transform even if output already existsskip existing
--package-id ID—Override packageId attribute in outputpreserve from source
--no-tests—Skip the test suite after transformationtests run
--open-folder—Open output folder in file manageroff
--quiet-qOnly show errors and final summaryverbose
05 — Test suite

test_transformation.py — Validate the Output

Runs 10 test groups with 30+ individual assertions. Checks every transformation operation: namespace rewrite, pubDate normalisation, personnel field fidelity, no empty elements, no 2.1.1 artefacts, and more.

Run against the bundled sample (quickest)

bash
$ python3 test_transformation.py

# Expected output:
# ━━ EML 2.1.1 → EML 2.2.0 — Transformation Test Suite ━━━━━━━━━━━━━━
#    Stylesheet : .../xslt/main.xsl
#    Input file : .../examples/input/sample-eml211.xml
# [0] Pre-flight checks
#   ✅  Input namespace confirmed as EML 2.1.1
# ...
# ━━ Results ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
#    Total: 30  │  Passed: 30  │  Failed: 0
#    🎉  All tests passed.

Run against your own EML 2.1.1 file

bash
$ python3 test_transformation.py \
    --input examples/input/your-dataset-eml211.xml

Test and save the output file

bash
$ python3 test_transformation.py \
    --input  examples/input/your-dataset-eml211.xml \
    --output examples/output/your-dataset-eml220.xml

Run against the Lesina real-world dataset

bash
$ python3 test_transformation.py \
    --input examples/input/lesina-phytoplankton-eml211.xml

# This runs 37 assertions against the full Lesina phytoplankton dataset.

Test all flags at a glance

FlagShortDescriptionDefault
--input FILE-iEML 2.1.1 input file to test againstexamples/input/sample-eml211.xml
--output FILE-oWrite the transformed result to this pathnot written
--xsl FILE—Use a custom stylesheet instead of xslt/main.xslxslt/main.xsl
--package-id ID—Override packageId in transformationpreserve from source
⚠ Important: Always pass an EML 2.1.1 file as input. If you accidentally pass an EML 2.2.0 file, the script detects it and warns you with a clear message instead of failing silently.
06 — CI validation

validate.sh — Batch CI Runner

Used in continuous integration pipelines (GitHub Actions, GitLab CI). Applies each stylesheet to its example inputs and diffs the result against the expected outputs.

Requirements: Java 8+ and Saxon-HE JAR. Set SAXON_JAR env variable to the JAR path.

Run all transformations

bash — from repo root
# From the repository root:
$ export SAXON_JAR=/path/to/saxon-he.jar
$ bash ci/validate.sh

Run one specific transformation

bash — from repo root
$ bash ci/validate.sh EML211_to_EML220
07 — Advanced usage

Advanced Commands

Run with Saxon (Java — recommended for production)

bash — Saxon-HE
# Basic transformation:
$ java -jar saxon-he.jar \
    -s:examples/input/your-dataset-eml211.xml \
    -xsl:xslt/main.xsl \
    -o:examples/output/your-dataset-eml220.xml

# With packageId override:
$ java -jar saxon-he.jar \
    -s:examples/input/your-dataset-eml211.xml \
    -xsl:xslt/main.xsl \
    -o:examples/output/your-dataset-eml220.xml \
    package-id=my.new.package.id

Run with xsltproc (Linux/macOS built-in)

bash — xsltproc (XSLT 1.0 only)
$ xsltproc xslt/main.xsl \
    examples/input/your-dataset-eml211.xml \
    > examples/output/result.xml

Use the Python API directly

python3 — inline script
$ python3 -c "
from lxml import etree

xsl    = etree.parse('xslt/main.xsl')
t      = etree.XSLT(xsl)
src    = etree.parse('examples/input/your-dataset-eml211.xml')
result = t(src)

with open('output.xml', 'wb') as f:
    f.write(etree.tostring(result.getroot(),
            pretty_print=True,
            xml_declaration=True,
            encoding='UTF-8'))
print('Done')
"

Push the repository to GitHub

bash — from repo root
# First time — connect to GitHub and push:
$ git remote add origin \
    https://github.com/your-org/lifewatch-xslt-transformations.git
$ git push -u origin main

# After making changes — stage, commit, push:
$ git add -A
$ git commit -m "feat: add new transformation for XYZ dataset"
$ git push

Extract one transformation as a standalone public repo

bash — from repo root
# Split EML211_to_EML220 into its own public repo:
$ git subtree split \
    --prefix=transformations/EML211_to_EML220 \
    -b split-eml211
$ git push \
    https://github.com/your-org/EML211_to_EML220.git \
    split-eml211:main

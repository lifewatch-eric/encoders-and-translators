LifeWatch XSLT / transformations / EML211_to_EML220
EML 2.1.1 → EML 2.2.0
✅ Stable XSLT 1.0 Saxon-HE · lxml MIT License

Upgrades an Ecological Metadata Language document from version 2.1.1 to 2.2.0. Handles nine distinct transformation operations: namespace rewrites, schema location updates, pubDate normalisation, distribution URL harvesting, structural reshaping of project and methods blocks, and a recursive identity copy for all unchanged content.

01 — Overview

What this stylesheet does

EML 2.2.0 introduced several breaking changes from 2.1.1: a new HTTPS namespace URI, a stricter content model for <dataset> requiring <alternateIdentifier> as its first child, an updated StMML schema reference, and more rigid ordering in <project> and <methods>. This stylesheet automates all of those changes in a single pass.

The stylesheet is written in XSLT 1.0 so it runs on any conformant processor — Saxon-HE, xsltproc, Xalan, or Python's lxml — without requiring XSLT 2.0 features.

03 — Operations

All Transformation Operations

The stylesheet performs nine distinct operations, executed in a single pass over the input document.

  • ①

    Namespace & Schema Location Update

    The root element is reconstructed as eml:eml bound to the EML 2.2.0 namespace https://eml.ecoinformatics.org/eml-2.2.0. The xsi:schemaLocation attribute value is rewritten using the recursive replace-string named template:

    FindReplace with
    eml://ecoinformatics.org/eml-2.1.1eml://ecoinformatics.org/eml-2.2.0
    http://www.xml-cml.org/schema/stmml-1.1http://www.xml-cml.org/schema/stmml-1.2
  • ②

    packageId Handling

    The stylesheet accepts an optional package-id parameter (default: 'newID'). When the parameter is not supplied (or equals the sentinel value), the existing packageId attribute is preserved unchanged. When a custom value is supplied, the attribute is overwritten.

    Saxon — override packageId
    java -jar saxon-he.jar -s:input.xml -xsl:main.xsl -o:output.xml \
  package-id=my.new.package.id
  • ③

    alternateIdentifier Injection

    EML 2.2.0 requires <alternateIdentifier> to appear as the first child of <dataset>. The stylesheet injects an empty <alternateIdentifier/> element unconditionally before any other dataset children. Downstream pipelines should populate this with a persistent identifier (DOI, handle, etc.).

  • ④

    Distribution URL Harvest

    A new <distribution> element is assembled inside <dataset> by collecting <online> blocks from two source paths:

    Source XPathExtracted fields
    //additionalMetadata/metadata/gbif/physicalobjectName → onlineDescription, distribution/online/url → url
    //dataset/otherEntity/physicalsame mapping as above
  • ⑤

    pubDate Normalisation

    EML 2.2.0 requires dates in ISO 8601 format. Many EML 2.1.1 records store only a four-digit year. The stylesheet detects year-only values by checking for the absence of a hyphen (-) and appends -01-01:

    Input valueOutput valueRule
    20192019-01-01no hyphen → append -01-01
    2021-03-042021-03-04hyphen present → pass through
  • ⑥

    dataTable Passthrough with Guard

    <dataTable> elements are copied with a defensive guard on <attributeList>/<attribute>: if attribute children exist, each is reshaped to ensure <attributeDefinition> is present. All other content is passed through the identity copy template unchanged.

  • ⑦

    Project Normalisation

    The EML 2.2.0 <project> content model requires a stricter element order. The stylesheet reshapes each <personnel> block:

    FieldBehaviour
    project/titleOnly the first title is emitted (EML 2.2.0 allows one)
    personnel/organizationNameSet to "Not available" when absent in source
    personnel/positionNamePopulated from <role>; falls back to "Not available"
  • ⑧

    Methods & Sampling Reshape

    Each <methodStep> is reconstructed. Multiple <para> elements inside a <description> are joined with a space separator into a single <para>. <instrumentation> and <software> sub-elements are also carried through. <sampling> blocks have their studyExtent and samplingDescription rewrapped.

  • ⑨

    Identity Copy (catch-all)

    All nodes not matched by a more specific template fall through to the identity copy template, which preserves namespace-qualified element names and all attributes unchanged. This ensures that any EML 2.1.1 element without a specific migration rule is carried through safely.

    xslt/main.xsl — identity template
    <xsl:template match="*">
  <xsl:element name="{name(.)}" namespace="{namespace-uri(.)}">
    <xsl:copy-of select="@*"/>
    <xsl:apply-templates/>
  </xsl:element>
</xsl:template>
04 — Running the transformation

Usage

Saxon-HE (recommended)

bash
# Basic run
java -jar saxon-he.jar \
  -s:examples/input/sample-eml211.xml \
  -xsl:xslt/main.xsl \
  -o:output.xml

# Override packageId
java -jar saxon-he.jar \
  -s:examples/input/sample-eml211.xml \
  -xsl:xslt/main.xsl \
  -o:output.xml \
  package-id=my.new.package.id

Python lxml (no Java needed)

python3
from lxml import etree

xsl_doc   = etree.parse("xslt/main.xsl")
transform = etree.XSLT(xsl_doc)
src_doc   = etree.parse("examples/input/sample-eml211.xml")

# default run
result = transform(src_doc)

# with packageId override
result = transform(src_doc, **{"package-id": etree.XSLT.strparam("my.id")})

# write output
with open("output.xml", "wb") as f:
    f.write(etree.tostring(result.getroot(), pretty_print=True,
            xml_declaration=True, encoding="UTF-8"))
05 — Automated testing

Testing

The bundled test_transformation.py script runs 10 test groups (41 individual assertions) against the transformation. It accepts any EML 2.1.1 file as input.

bash
# Test against bundled sample
python3 test_transformation.py

# Test against your own EML 2.1.1 file
python3 test_transformation.py \
  --input /path/to/your-eml211.xml \
  --output /path/to/result.xml

# CI — from repo root
bash ci/validate.sh EML211_to_EML220
⚠ Important: The test script performs a pre-flight check to confirm the input file is EML 2.1.1. If you pass an EML 2.2.0 file by mistake, the script will warn you clearly rather than producing incorrect results silently.
06 — Known limitations

Known Limitations

LimitationReason / workaround
<alternateIdentifier> always injected empty Downstream systems must populate with a persistent identifier (DOI, handle)
Year-only pubDate normalised to YYYY-01-01 Month/day precision cannot be inferred; manually correct if known
XSLT 1.0 — no fn:replace() String substitution uses a recursive named template; works on all processors
EML 2.2.0-only elements not back-populated <annotation>, <licensed>, <unitDictionary> are not in EML 2.1.1 source; add manually
